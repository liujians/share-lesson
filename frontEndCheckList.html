Head
HTML
  1. Clean up comments（清除注释）
  2. 所有页面都在浏览器上通过测试(Safari, Firefox, Chrome, Internet Explorer, EDGE
  3. 页面中链接没有断开，请确认你没有404错误。
链接检查器
CSS
  1. 使用css预处理器(Sass Less)
  2. 如果使用了ID，确保ID的唯一性。
  3. 使用CSS reset(如reset.css, normalize.css)
  4. 对部分属性加上浏览器内核前缀，生成你浏览器兼容的属性。(autoprefixer)
  1. 压缩所有CSS文件
  2. 检查css的style-lint

JavaScript
  1. 确保没有任何JS代码内联(与HTML代码混合)。
  2. 压缩所有js文件(可以添加.min后缀)。
  3. 用ESLint检测并没有错误(基于你的配置规则)。
  4. 非阻塞JS(使用async或defer异步加载)

defer和async的区别 浅谈script标签的defer和async
Security
  1. 确保向服务器端发出的请求是合法的，并来自您的网站/应用程序，以防止发生CSRF攻击。
  2. 页面或网站没有XSS攻击的可能性。

Performance
  1. 每张网页的大小在0到500KB之间
  2. 压缩你的HTML文件。
  3. 图片、js脚本和CSS需要懒加载
web speed test
Accessibility
  1. 所有页面都有H1
  2. 标题应以正确的顺序合理使用(H1至H6)。
  3. 颜色对比度至少通过WCAG AA标准
颜色对比工具
Landmarks
  1. <header> 标签中加入 role="banner"属性。
  2. <nav> 标签中加入 role="navigation"属性。
  3. <main> 标签中加入 role="main"属性。
  4. label与每个输入表单元素相关联，如果label无法显示，请使用aria-label代替。
